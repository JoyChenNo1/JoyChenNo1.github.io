飞书等第三方渠道的账号同步

发表于2022-02-13|更新于2025-03-19|job

背景

工作中会涉及到飞书、钉钉、AAD(Azure Active Directory) 及第三方客户 OA 系统的对接，主要用于同步用户组织架构到业务系统中，同步原理基本类似，基于组织架构下的人员同步，根据第三方系统提供的API拉取组织架构和人员列表，根据人员的手机或者邮箱等信息进行业务系统成员的同步。下面以飞书账号同步作为阐述。

同步方案设计

设计要点：

1、crontab 定时任务

2、gin 框架的使用

3、业务接口调用，避免直接操作数据库表

4、组织架构树的构建

5、同步类的接口设计

6、优雅关闭

7、viper 实现配置读取，cmd 形式（cobra）实现多种不同的同步方式。


package main

func main() {
  // common.InitConfig()
  
  db.Init()
  
  sync.FeishuSync().Init(ctx)
  
  crobtab.StartCrontab(ctx)
  
  // http server listen for feishu callback to update app_ticket info
  serv := http.New(common.Config.Http.Addr)
  herewe.Go(func(){
    serv.ListenAndServe()
  })
  
  // fully sync user accounts
  sync.FeishuSync().Run(ctx)
  
  
  // graceful exit
  GracefullyExit(func(){
    cancel()
    serv.Close()
  })
}

// GracefullyExit to release program resources and exit gracefully
func GracefullyExit(exitFunc func()) {
  c := make(chan is.Signal, 1)
  
  signal.Notify(c, syscall.SIGHUP, syscall.SIGINT, syscall.SIGERM, syscall.SIGQUIT)
  for s := range c {
    switch s {
    case syscall.SIGHUP, syscall.SIGINT, syscall.SIGERM, syscall.SIGQUIT:
    	// exit by signal
      
      // before exit do some work
      exitFunc()
      
      time.Sleep(time.MillionSecond * time.Duration(1000))
      os.Exit(0)
      
    default:
      // unexspectedd exit by unknown signal
      
      exitFunc()
      time.Sleep(time.MillionSecond * time.Duration(1000))
      os.Exit(0)
      
    }
  } 
}
