C++17 新特性

本篇章主要是根据《C++17 STL Cookbook》以及 https://dl.acm.org/doi/10.1145/3386320 中，Bjarne Stroustrup 对 C++ 特性文章 《Thriving in a crowded and changing world: C++ 2006--2020》 的阐述。



Class template argument deduction（类模板参数推导）

C++17 能让 [编译器] 自动的从所调用的构造函数，推导出模板类型。

std::make_shared 、std::make_unique、std::make_pair、std::make_tuple 这些工厂函数在 C++17中过时了。

// C++17 之前
std::pair <int, char> my_pair(1, '1');
std::tuple<int, double, const char*> my_tuple(123, 12.3, "abc");
// C++17
std::pair my_pair(1, '1');
std::tuple my_tuple(123, 12.3, "abc");

当然，还存在一些不能依赖类型推导的情况，在此例子开始之前，先引入模板中推导指引的概念。

[注]：下面涉及到的 推导指引（Deduction Guides）的内容，详情可参考《C++ Template snd Edition》--David Vandevoorde, Nocolai M, Josuttis, Douglas Gregor.第二章模板类型推到关于 Deduction Guides 的介绍。

推导指引：提供额外的模板参数推断规则，或者修正已有的模板参数推断规则。

比如，你可以定义，当传入一个 C 风格字符串时，用 std::string 实例化 Stack 模板类：

namespace pdfCore {
template <typename T>
class Stack {
private:
       std::vecotr<T> elements_;
public:
       Stack(T es) : elements_(es) {...}
       ...
};
// 指引语句必须出现在和模板类定义的同一个作用域或者 namespace 里
Stack( const char* ) -> Stack<std::string>; // ->后面的类型被称为 "guided type"

} // namespace pdfCore

不能依赖类型推导的情况：

#include <iostream>
#include <string>

#define LOG(x) std::cout << x << std::endl;

template <typename T>

class Sum {

public:
       // 实例化模板的时候，不显式指定，由编译器推导
       T value_;

       // 构造函数接受任意数量参数，并想加起来赋给 成员 value_
       template <typename ... Ts>
       Sum(Ts&&... values) : value_ { (values + ...) } {

       }
};

// 推导指引（Deduction Guides），告诉编译器用 std::common_type_t 的特性，
// 能找到适合所有值的共同类型
template <typename ... Ts>
Sum(Ts&& ... ts) -> Sum<std::common_type_t<Ts...>>;

int main() {
       // 不同类型参数
       Sum value { 1u, 2.0, 3, 4.0f };
       LOG(value.value_);

       // std::string 和C 风格字符串
       Sum str { std::string("Hello "), "World!"};
       LOG(str.value_);

       return 0;
}
$ ./default_deduction 
10
Hello World!


Fold expressions （折叠表达式）

类型 语法
一元右折叠 ( 形参包 op ... )
一元左折叠 ( ... op 形参包 )
二元右折叠 ( 形参包 op ... op 初值 )
二元左折叠 ( 初值 op ... op 形参包 )
op - 任何下列 32 个二元运算符之一：+ - * / % ^ & | = < > << >> += -= = /= %= ^= &= |= <<= >>= == != <= >= && || , . ->*。在二元折叠中，两个 op 必须相同。
形参包 - 含未展开的形参包且其顶层不含有优先级低于转型（正式而言，是 转型表达式）的运算符的表达式
初值 - 不含未展开的形参包且其顶层不含有优先级低于转型（正式而言，是 转型表达式）的运算符的表达式

注意开与闭括号是折叠表达式的一部分。

参数包的位置即代表左折叠 (ts + ...) 还是右折叠 (... + ts)。

#include <iostream>
#include <string>

#define LOG(x) std::cout << x << std::endl
#define STR(str) std::string {str}

template <typename ... Ts>
auto Add(Ts ... ts) {
    return (ts + ... );
   }

int main() {
    int add_int = Add(1, 2, 3, 4, 5);
    LOG(add_int);
    std::string result = Add( STR("how "), STR("are "), STR("you "));
       LOG(result);
       return 0;
}
$ ./fold_add 
15
how  are you

有了折叠表达式，类型如下的例子功能实现变得简洁许多：

#include <iostream>
#include <set>
#include <algorithm>

#define LOG(x) std::cout << x << std::endl


// 统计 string / vector / set / list array 等容器中，
// 包含某些元素的个数情况
template <typename R, typename ... Ts>
auto CountContains(const R& container, Ts ... ts) {
       return (std::count(std::begin(container), std::end(container), ts) + ...);
}

// 对 set 实现批量插入功能
// 当参数包的某个元素插入失败，根据 && 远算符的短路特性，
// 会总之插入剩余元素
template <typename R, typename ... Ts>
bool InsertAll(R &set, Ts ... ts) {
       return (set.insert(ts).second && ...);
}

// 检查给定的多个变量是否在某个范围内
template <typename R, typename ... Ts>
bool IsAllWithin(R min, R max, Ts ... ts) {
       return ( (min <= ts && ts <= max) && ... );
}

int main() {
    std::string str {"aabbccddeeffgg"};
       int count_abc = CountContains(str, 'a', 'b', 'c');
       LOG(count_abc);
   
    std::set<int> my_set = { 1, 2, 3 };
       InsertAll(my_set, 4, 5, 6);
       LOG("set 批量插入元素后 ： ");
       std::for_each(my_set.begin(), my_set.end(), [](const auto& item){
           LOG(item);
       });
   
    return 0;
   }

Structured bindings （结构化绑定）

auto [var1, var2, ...] = < pair, tuple, struct, 固定长度数组>

注意个数要匹配。

和 std::tie 相比，std::tie 进行赋值的时候，所有变量需要提前定义，但是可以使用 std::ignore 作为虚拟变量。所以，要根据实际需求选择使用。

#include <iostream>
#include <tuple>
#include <string>
#include <vector>

std::pair<int, int> Float2Int(float a, float b) {
       return std::make_pair<int, int>(int(a), int(b));
}

std::tuple <std::string, int, char> Info() {
       return std::make_tuple<std::string, int, char>("李丽", 18, 'S');
}

//打印信息
template<typename T>
void log(T t) {
       std::cout << t << std::endl;
}

template<typename T, typename... Args>
void log(T t, Args...args) {
    std::cout << t << std::endl;
    log(args...);
}

struct student {
    std::string name;
    int age;
    char grade;
};

int main() {
    const auto result = Float2Int(1.2f, 2.3f);
    log("pair 形式: ");
    log(result.first, result.second);

    auto [a, b] = Float2Int(13.0f, 16.05f);
    log("pair 结构化绑定: ");
    log(a, b);

    const auto [name, age, grade] = Info();
    log("tuple 结构化绑定: ");
    log(name, age, grade);

    // 初始化一个结构体列表
    std::vector<student> class_one = {
        {"Xiaoming", 21, 'A'},
        {"Xiaohong", 20, 'S'},
        {"DaWei", 22, 'B'},
    };
    
    log("结构化绑定的方式遍历列表: ");

    // 可以是 auto, const auto, const auto&, auto&&，适当使用引用，减少不必要的拷贝。 
    for (const auto& [name, age, grade] : class_one) {
        log( name, age, grade);
    }
    
    return 0;
}

将变量作用于限制在 if 和 switch 内

#include <iostream>
#include <map>
#include <algorithm>

int main (void) {
    std::map<int, char> mapper = {
        {1, '1'},
        {2, '2'},
        {3, '3'}
    };

    if(auto result(mapper.find(3)); result != mapper.end()) {
        std::cout << "map 存在 key (3)." << std::endl;
    }


    switch (auto ch = getchar(); ch) {
        case 'a': 
            std::cout << "左转" << std::endl;
            break;
        case 'd': 
            std::cout << "右转" << std::endl;
            break;
        case 'w': 
            std::cout << "前进" << std::endl;
            break;
        case 's': 
            std::cout << "后退" << std::endl;
            break;
    }
}

variant

variant<T, U> : holds a T or a U

#include <iostream>
#include <string>
#include <variant>

// 定义具有3个重载operator() 的 functor
struct SOutput {
	void operator()(const int& i) {
		std::cout << i << std::endl;
	}

 	void operator()(const double& d) {
		std::cout << d << std::endl;
	}

	void operator()(const std::string& s) {
 		std::cout << s << std::endl;
	}
};


int main() {

    // 第一可以是int/double/string的 vriant y
    std::variant<int, double, std::string> y;

       // 赋值为 int
       y = 1;
        std::visit(SOutput(), y);
   
       // 赋值为 double
       y = 23.0;
    std::visit(SOutput(), y);
   
       // 赋值为 string
       y = "hello";
    std::visit(SOutput(), y);
   
       // 可以使用 std::get获取值，但是要对应上，比如现在y是string
       // 类型不对会抛出异常 std::bad_variant_access e
    std::string str = std::get<std::string>(y);
   
       std::cout << str << std::endl;
   
    // 也可以用 std::get_if
       int* value = std::get_if<int>(&y);
    if (!value) {
           std::cout << "类型不匹配！" << std::endl;
       } else {
           std::cout << "正确取值：" << *value << std::endl;
       }
   
       return 0;
   }

#include <variant>
#include <string>
#include <iostream>
#include <vector>

template<class ... Ts> 
struct CustomOutput : Ts ... { 
    using Ts::operator()...; 
};

template<class ... Ts> 
CustomOutput( Ts ...) -> CustomOutput < Ts ... >;


void Use() {
       //  定义 variant 类型的 vector
       using var_t = std::variant < int, long, double, std::string >;
       std::vector<var_t> vec = {10, 20L, 30.40, "hello"};

       CustomOutput out_put = {
           // 输出 int, long
           [](auto arg) { std::cout << arg << std::endl; },
           // 输出 double
           [](double arg) { std::cout << arg << std::endl; },
           // 输出 string
           [](const std::string& arg) { std::cout << arg << std::endl; },
       };

       // 遍历
       for ( auto & var : vec ) {
           std :: visit(out_put, var );
       }
}

int main() {
       Use();
       return 0;
}

{} 的初始化规则

不允许使用：auto var {v1, v2, v3, ...}

相比另一种合法使用的方式：auto var = {v1, v2, v3, ...} 因为这会调用std::initializer_list<T>进行初始化。

其中，v1, v2, v3, ... 必须是同等类型 T。

// 对于初始化，{} 与 () 调用构造函数初始化的方式，
// 不同点在于 {} 没有类型的隐式转换
auto var1(3.00f);
auto var2{3.0f};
std::cout <<  typeid(var1).name()  << std::enndl; // i
std::cout << typeid(var2).name() << std::enndl;; // f

对于只支持 C++11、C++14 的编译器，auto x{ 123 } 的类型可能会推导成整型，也可能会推导成std::initializer_list<int>.

所以会存在可移植性的问题。



inline variables （内联变量）

解决对于库文件 .hpp 中 定义了一个静态变量，这个头文件被多个 .cpp包含后链接阶段报错问题。

// xxx.hpp
class Rect {
public:
    static const int x = 10;
};

// yyy.cpp 
include "xxx.hpp"

// zzz.cpp
include "xxx.hpp"

C++17 之前为了修复类似的问题， 会这么做：

// xxx.hpp
class Rect {
    static const int& GetX() {
        static const int x = 10;
        return x;
    }
};

为了修复这个问题，添加 inline 关键字，从而避免符号重定义问题：

// xxx.hpp
class Rect {
public:
    static const inline int x = 10;
};


constexpr-if

constexpr-if 的工作机制与传统的 if-else 类似。不同点在于，constexpr-if 是编译时进行判断，if-else 是运行时判断。

使用 constexpr-if 的代码编译完成后，程序的这一部分代码不会有分支存在，这种方式其实类似于 #if-#else的预编译方式进行宏替换，但是，constexpr-if 更优雅。

正如 《C++17 in Detail》里所说，All the complicated syntax of std::enable_if went away..., if constexpr evaluates the condition and only one block will be compiled.

#include <iostream>
#include <vector>

#define LOG(x); std :: cout << x << std::endl;

template <typename T>
class Addable {

private:
       T val_;

public:
       Addable(T v) : val_(v) { }

       const auto& value() { return val_; }

       template <typename U>
       T add(U x) {
           if  constexpr(std::is_same<T, std::vector<U>>::value) { 
               auto copy {val_};
               for (auto& item : copy) {
                   item += x;
               }
               return copy;
           } else {
               return val_ + x;
           }
       }

};

int main() {
       // 实例化不同类型的 Addable 对象，调用加法函数。
       LOG(Addable<int> {1}.add<int>(2));

       Addable<std::vector<int>> iv {{1, 2}};
       iv.add<int>(3);
       for(const auto& item : iv.value()) {
           LOG(item);
       }
       return 0;
}
$ ./constexpr_if 
3
1
2
