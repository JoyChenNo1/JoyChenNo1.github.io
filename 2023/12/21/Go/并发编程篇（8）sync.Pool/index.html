Go 并发编程篇（8）Pool

发表于2023-12-21|更新于2025-03-18|golang

前言

鸟窝的《深入理解 Go 并发编程》读书笔记以及其他go并发编程知识学习的汇总，持续完善中。


sync.Pool

用法

package main

import (
    "fmt"
    "io"
    "net/http"
    "sync"
)

// 定义一个全局的 http.Client 池
var httpClientPool = sync.Pool{
    // 当池为空时，使用 New 函数创建新的 http.Client 实例
    // 使用时，一定要指定 New 方法，因为在并发量不大的时候，GC 有可能回收 Pool 中已经创建的对象
    New: func() interface{} {
        return &http.Client{
          Timeout: 5 * time.Second,
        }
    },
}

func main() {
    // 从池中获取一个 http.Client 实例
    client := httpClientPool.Get().(*http.Client)
    defer httpClientPool.Put(client)

    // 发起 HTTP 请求
    resp, err := client.Get("https://www.example.com")
    if err != nil {
        fmt.Println("Error:", err)
        return
    }
    defer resp.Body.Close()

    // 读取响应内容
    body, err := io.ReadAll(resp.Body)
    if err != nil {
        fmt.Println("Error reading response:", err)
        return
    }

    fmt.Println("Response body:", string(body))
}

实现

  • 本地池和共享池：每个 P（处理器）都有一个本地池，用于存储该 P 上的对象。此外，还有一个共享池，用于存储所有 P 共享的对象。当从对象池中获取对象时，会先尝试从本地池获取，如果本地池为空，则从共享池获取。当将对象放回对象池时，会先尝试放回本地池，如果本地池已满，则放回共享池。
  • 垃圾回收清理：在每次 GC 时，当并发不大时，sync.Pool 有可能会清空空闲的对象池，以避免对象池中的对象占用过多的内存。

local：是一个指向 [P]poolLocal 数组的指针，其中 P 是 Go 运行时的处理器数量。每个 poolLocal 实例对应一个处理器，用于存储该处理器私有的对象。

localSize： 表示 local 数组的大小。

victim 和 victimSize：在 GC 时，local 中的对象会被移动到 victim 中，用于实现对象池的清理和回收。

New：是一个函数指针，当对象池为空时，会调用该函数创建新的对象。 poolLocalInternal：包含两个字段，private 存储每个处理器私有的对象，shared 是一个链表，存储多个处理器共享的对象。

type Pool struct {
    noCopy noCopy

    local     unsafe.Pointer // 指向 [P]poolLocal 数组的指针
    localSize uintptr        // local 数组的大小

    victim     unsafe.Pointer // 上一轮垃圾回收时的 local
    victimSize uintptr        // victim 数组的大小

    New func() interface{} // 当对象池为空时，用于创建新对象的函数
}

type poolLocal struct {
    poolLocalInternal

    // 用于静态存储每个 P 的 poolLocal 实例
    pad [128 - unsafe.Sizeof(poolLocalInternal{})%128]byte
}

type poolLocalInternal struct {
    private interface{} // 每个 P 私有的对象
    shared  poolChain   // 多个 P 共享的对象链表
}

1、调用 p.pin() 函数将当前 goroutine 固定到一个处理器上，并获取该处理器对应的 poolLocal 实例。

2、首先尝试从 poolLocal 的 private 字段中获取对象，如果有则返回，并将 private 字段置为 nil。

3、如果 private 字段为空，则尝试从 shared 链表中获取对象。

4、如果 shared 链表也为空，则调用 p.getSlow(pid) 函数从其他处理器的 shared 链表中窃取对象。

5、如果仍然没有获取到对象，且 New 函数不为 nil，则调用 New 函数创建一个新的对象。

func (p *Pool) Get() interface{} {
    l, pid := p.pin()
    x := l.private
    l.private = nil
    if x == nil {
        // 从共享链表中获取对象
        x, _ = l.shared.popHead()
        if x == nil {
            x = p.getSlow(pid)
        }
    }
    runtime_procUnpin()
    if x == nil && p.New != nil {
        x = p.New()
    }
    return x
}
  1. 调用 p.pin() 函数将当前 goroutine 固定到一个处理器上，并获取该处理器对应的 poolLocal 实例。
  2. 如果 poolLocal 的 private 字段为空，则将对象放入 private 字段。
  3. 如果 private 字段不为空，则将对象放入 shared 链表的头部。
func (p *Pool) Put(x interface{}) {
    if x == nil {
        return
    }
    l, _ := p.pin()
    if l.private == nil {
        l.private = x
        x = nil
    }
    if x != nil {
        l.shared.pushHead(x)
    }
    runtime_procUnpin()
}
